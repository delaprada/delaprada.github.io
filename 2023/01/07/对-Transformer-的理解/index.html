对 Transformer 的理解

Transformer 架构

Transformer 由 Encoder 和 Decoder 构成。

Encoder 是由 N 个相同的 layer 构成（N 为 6），每个 layer 有两个子 layer，第一个子 layer 是 multi-head self-attention，第二个子 layer 是 simple, position-wise fully connected feed-forward network。每个子 layer 都应用了 residual-connection（resnet），然后接着应用 Layer Normalization。

Decoder 同样是由 N 个相同的 layer 构成（N 为 6），除了在 Encoder 中出现的两个子 layer 以外，Decoder 还多增加了一个子 layer，用于对 Encoder 的输出做 masked multi-head self-attention。

在 masked multi-head self-attention 机制下，当前位置的 query 只会与之前位置的 key 做 self-attention，不会与之后位置的 key 做 self-attention。Transformer 这么做的原因是，对于 sequence model 来说（比如：语音识别），输出是一个一个产生的，所以它只能够考虑到在当前输出之前的信息。

Q K V 是什么

每个 input a’ 会对应生成 query, key, value，query 是 input a’ 乘 matrix Wq , key 是 input a’ 乘 matrix Wk , value 是 input a’ 乘 matrix Wv , 这三个 matrix 是不一样的，在训练过程中每个 matrix 的参数会不断更新。

output 是 weighted sum of values，每个 weight 是通过 query 和对应 key 点乘再经过 softmax 得到，也可以称为是 attention score。因此，与当前 input 越相近的 input（即 attention score 越高），就会主导当前的 output。

dot product（内积）

每个 query 和 key 之间进行的操作是 dot product，那么什么是 dot product，为什么要做 dot product 呢？

dot product 指的是内积，它有两种定义方式，一种是代数方式，另一种是几何方式。

从代数角度出发

设二维空间内有两个向量 $\overrightarrow{a} = (x_1, y_1)$ 和 $\overrightarrow{b} = (x_2, y_2)$ ，定义它们的数量积（又叫内积、点积）为以下实数：

$$ \overrightarrow{a} \cdot \overrightarrow{b} = x_1x_2 + y_1y_2 $$

更一般地，n维向量的内积定义如下：

$$ \overrightarrow{a} \cdot \overrightarrow{b} = \sum\limits_{i=1}^n a_ib_i = a_1b_1 + a_2b_2 + a_3b_3 + … + a_nb_n $$

从几何角度出发

设二维空间内有两个向量 $\overrightarrow{a} = (x_1, y_1)$ 和 $\overrightarrow{b} = (x_2, y_2)$，$\mid a \mid$ 和 $\mid b \mid $ 表示向量 a 和 b 的大小，它们的夹角为 $\theta$ ( $0 <= \theta <= \pi$) ，则内积定义为以下实数：

相当于 a 向量在 b 向量投影：

$$ \overrightarrow{a} \cdot \overrightarrow{b} = \mid\overrightarrow{a}\mid \mid \overrightarrow{b} \mid cos\theta $$

positional Encoding（位置编码）

什么是 Positional Encoding

在任何一门语言中，词语的位置和顺序对句子的意思表达是至关重要的。传统的 RNN 模型在处理句子的时候，会以序列的模式逐个处理句子中的词语，这使得词语的顺序信息在处理过程中被天然的保存下来，并不需要额外的处理。

而对于 Transformer 来说，由于句子中的词语是同时进入网络进行处理的，顺序信息在输入网络时就已经丢失了。因此，Transformer 是需要额外的处理告知每个词语其相对位置是什么。Attention is all you need 这篇文章就使用了 Positional Encoding 将位置信息编码添加到输入中，让网络模型知道每个词的位置和顺序。

应该如何做 Positional Encoding

一开始我们能想到的方法，是给句子中的每个词赋予一个相位，也就是 [0, 1] 中间的一个值，第一个词是 0，最后一个词是 1，中间的词在 0 到 1 之间取值。但这种方法的一个问题在于，你不知道你要在这个特定的范围 [0, 1] 之间表达多少个词。换而言之，对于不同长度的句子来说，词语之间的间隔表达的含义就不一致。

另一种可以想到的方式是，用线性增长的方式来进行位置编码。比如，第一个词的位置是 0，第二个词的位置是 1，以此类推。但这样的问题在于，位置编码的取值可能很大，整个模型所接受的序列也会更长。模型的泛化性能也会减弱。

因此，理想情况下，编码方式要满足以下标准：
1.对于句子中处在不同位置上的词，要赋予独一无二的位置编码
2.在不同长度的句子中，两个词语间的间隔所代表的含义是一致的
3.模型能够简便地延伸到更长的句子。

因此，Attention is all you need 这篇文章提出了一个满足以上标准的位置编码方式。
