对回溯算法的一点新认识

前言

昨天刷题的时候，恰好刷到了两道与回溯相关的题目。之前做题的时候遇到和回溯算法相关的题都很怕，主要刷题量不够且没有掌握回溯的规律和解题方法。在看到评论区大佬的一些解析后觉得很有感悟，想借此总结一下 🙌。

回溯算法的使用场景

回溯算法经常被应用于要求所有组合或者符合条件的所有组合的题目中。

回溯算法的核心是回溯。在搜索到某一节点的时候，如果我们发现目前的节点并不是需求目标时，我们回退到原来的节点继续搜索，并且把在目前节点修改的状态还原。

具体的写法的步骤为：[修改当前节点状态]->[递归子节点]->[回改当前节点状态]。

换句话说就是：

  1. 引用传状态
  2. 所有的状态修改在递归完成后回改

回溯算法与深度优先搜索

当回溯用于树的时候，就是深度优先搜索。其实几乎所有可以用回溯解决的问题都可以表示为树。那么回溯和深度优先搜索在这里就几乎同义了。

如果一个问题解决的时候显式地使用了树，那么我们就叫它dfs。

很多时候没有用树我们也管它叫dfs严格来说是不对的，实际上两者都可以剪枝，所以用剪枝来区分回溯和深搜是不对的。

深搜的题目可以分为主函数和辅函数，主函数用于遍历所有的搜索位置，判断是否可以开始搜索，如果可以即在辅函数进行搜索。

如果没有规定说哪些位置不可以进行搜索，那主函数就用于直接传参并调用辅函数。

回溯的模板

回溯算法的公式：

步骤如下：

  1. 找结束递归的条件
  2. for循环遍历
  3. 做选择并递归
  4. 撤销选择，即回溯（回到上一步）

例一：全排列

题目链接：46. 全排列

用回溯的模板解一下这题：

  1. 递归结束的条件：排列数组的长度为nums.length的时候，将排列数组push到结果数组中，然后return。
  2. for循环遍历：遍历nums数组。
  3. 做选择并递归：如果当前遍历到的数已经在排列数组中，则continue跳过该数；如果不在，则push进排列数组，然后进行递归。
  4. 撤销选择：用pop方法将刚刚push进排列数组的数删掉（不要傻傻用splice😥，更不要用slice😓。slice是创建一个新的数组，并不会原地修改数组本身。）

这题也可以创建一个visited数组用于记录遍历到的这个数是否已在数组中，但这里更简单的操作是用js的includes方法判断。

例二：组合总数

题目链接：39. 组合总和

用回溯的模板解一下这题：

  1. 递归结束的条件：组合数组的和为target时，将组合数组push进结果数组，然后return；当组合数组的和大于target时，直接return。
  2. for循环遍历：遍历candidates数组。
  3. 做选择并递归：将当前遍历到的数push进组合数组中，递归，递归时注意要传入当前遍历到的i值（组合和排列不同，组合的话不需要考虑顺序）。
  4. 撤销选择：用pop方法将刚刚push进组合数组的数删掉。

例三：电话号码的字母组合

题目链接：17. 电话号码的字母组合

这题的不同点在于，它的遍历对象不是一个数组，因为每个数字都对应一系列字母，相当于一个字母数组，所以是对多个数组进行组合。

和上题类似，我们需要在递归中传入一个pos变量，用于记录遍历到哪个字母组合。

用回溯的模板解一下这题：

  1. 递归结束的条件：当pos的值等于digits.length时，表明tmp字符串长度已为digits.length，将tmp字符串push进结果数组中，然后return。

    这里没有将tmp作为参数传入递归中的原因是，字符串只是基本数据类型，而不是引用数组类型，那么每次递归的tmp并不是同一个tmp，无法实现回溯，回溯是要基于同一个变量进行操作，上面的题目中传入递归的参数都是数组，数组是引用数组类型，可以实现。

    所以tmp要作为一个作用域变量，并将回溯的函数在这个作用域内定义。

  2. for循环遍历：遍历当前的字母组合。

  3. 做选择并递归：tmp字符串拼接当前遍历到的字母，递归，传入pos+1，表示遍历下一个字母组合。

  4. 撤销选择：用slice方法删除刚刚拼接的字母。

总结
