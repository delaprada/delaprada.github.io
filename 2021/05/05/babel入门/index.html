babel入门

简介

Babel是一个JS编译器，用于将ES6及以上的JS代码转换为向后兼容的JS语法，以便能够运行在当前和旧版本的浏览器或其他环境中（涉及新语法的转换和缺失特性的修补）。

  1. 安装所需的包

  2. 在项目根目录创建一个名为babel.config.json的配置文件（以下仅为示例）

  3. 如果使用的是Babel旧版本，则文件名为babel.config.js

  4. 运行命令将src目录下的所有代码编译到lib目录

    你可以利用 npm@5.2.0 所自带的 npm 包运行器将 ./node_modules/.bin/babel 命令缩短为 npx babel。

  • babel的核心功能在@babel/core模块中
  • @babel/cli是一个能够从终端（命令行）使用的工具（比如上面的--out-dir参数）。可以通过--help参数来查看命令行工具所能接受的所有参数列表，最重要的是--plugins和--presets这两个参数

插件（plugins）和预设（preset）

plugins

babel的代码转换功能是以插件的形式出现的。插件是小型的JS程序，用于指导babel如何对代码进行转换。

例如将ES6及以上的语法转换为ES5语法，可以使用诸如@babel/plugin-transform-arrow-functions之类的官方插件。

preset

因为如果每次都要开发者一个个添加并安装插件，配置文件会很长，并且安装时间也会很长。

preset是一组预先设定的插件。我们可以根据自己所需要的插件组合创建一个自己的preset并将其分享出去。

preset包括：env、react、flow、minify等。这里最重要的是env。

在执行顺序方面，原则是：

  • plugin会在preset之前运行
  • plugin会从前往后顺序执行
  • preset的顺序则正好相反（从后向前）

env

env是最为常用也是最重要的。env的核心目的是通过配置得知目标环境的特点，然后只做必要的转换。

例如目标浏览器是支持ES2015的，那么就不需要ES2015这个preset，于是代码就可以小一点（一般转化后的代码总会更长），构建时间也可以缩短一些。

配套工具

babel-polyfill

babel默认只转换js语法，而不转换新的API，比如Iterator、Generator、Set、Map、Proxy、Reflect、Symbol、Promise等全局对象，以及一些定义在全局对象上的方法（比如Object.assign）都不会转码。

比如，ES2015在Array对象上新增了Array.from方法，babel不会转码这个方法。如果想让这个方法运行，必须使用babel-polyfill（内部集成了core-js和regenerator）。

使用时，在所有代码运行之前增加 require('babel-polyfill')。或者更常规的操作是在 webpack.config.js 中将 babel-polyfill 作为第一个 entry。因此必须把 babel-polyfill 作为 dependencies 而不是 devDependencies。

但babel-polyfill有两个主要的缺点：

  1. 使用babel-polyfill会导致打出来的包非常大
  2. babel-polyfill会污染全局变量，给很多类的原型链作了修改。如果我们开发的也是一个类库供其他开发者使用，这种情况就会变得非常不可控。

在实际使用中，如果我们无法忍受这两个缺点（通常是第二个），我们会倾向于使用babel-plugin-transform-runtime。

babel-runtime和babel-plugin-transform-runtime

我们时常在项目中看到 .babelrc 中使用 babel-plugin-transform-runtime，而 package.json 中的 dependencies (注意不是 devDependencies) 又包含了 babel-runtime，其实两者是需要成套使用的。

babel-plugin-transform-runtime可以将定义方法改成引用，那么重复定义就能够转换成重复引用，就不存在代码重复的问题了。

babel-runtime就是这些定义方法的集合处，因此，在使用 babel-plugin-transform-runtime 的时候必须把 babel-runtime 当做依赖。

babel-runtime，它内部集成了

  1. core-js: 转换一些内置类 (Promise, Symbols等等) 和静态方法 (Array.from 等)。绝大部分转换是这里做的。自动引入。
  2. regenerator: 作为 core-js 的拾遗补漏，主要是 generator/yield 和 async/await 两组的支持。当代码中有使用 generators/async 时自动引入。
  3. helpers, asyncToGenerator 就是其中之一，其他还有如 jsx, classCallCheck 等等，可以查看 babel-helpers。

babel-plugin-transform-runtime 不支持 实例方法 (例如 [1,2,3].includes(1))
